# 从iOS越狱插件到Loon插件转换指南

## 🔄 转换原理

### iOS越狱插件 vs Loon插件

#### iOS越狱插件 (dylib)
```
工作原理：
1. 使用MobileSubstrate/CydiaSubstrate框架
2. 在应用启动时注入dylib动态库
3. 通过MSHook修改应用内的方法返回值
4. 直接修改内存中的数据

优点：
✅ 可以修改本地验证逻辑
✅ 可以Hook任何Objective-C方法
✅ 不需要网络请求
✅ 功能更强大

缺点：
❌ 需要越狱设备
❌ 需要重新编译和签名
❌ 应用更新后可能失效
❌ 安全性较低
```

#### Loon插件 (Script)
```
工作原理：
1. 使用MITM中间人攻击
2. 拦截HTTP/HTTPS请求
3. 修改服务器返回的响应数据
4. 返回修改后的数据给应用

优点：
✅ 不需要越狱
✅ 配置简单
✅ 易于分享和更新
✅ 相对安全

缺点：
❌ 只能修改网络请求
❌ 无法修改本地验证
❌ 需要服务器返回VIP状态
❌ 可能被证书绑定绕过
```

## 📊 转换流程对比

### 原始dylib插件分析

#### 从dylib中提取的信息：
```
1. 目标应用: com.abishkking.maleworkout
2. Hook方法: 
   - isVip
   - isPro
   - isPremium
   - hasValidSubscription
3. 功能:
   - 解锁VIP功能
   - 清理缓存和数据
   - Keychain操作
```

#### dylib的Hook逻辑：
```objective-c
// 原始dylib中的Hook代码（推测）
MSHookMessageEx(
    objc_getClass("UserProfile"),
    @selector(isVip),
    (IMP)&hooked_isVip,
    (IMP*)&original_isVip
);

BOOL hooked_isVip(id self, SEL _cmd) {
    return YES; // 直接返回true
}
```

### Loon插件实现

#### 对应的Loon脚本：
```javascript
// Loon脚本中的修改逻辑
let obj = JSON.parse($response.body);
obj.data.isVip = true; // 修改响应数据
$done({body: JSON.stringify(obj)});
```

## 🔍 转换步骤详解

### 步骤1: 分析dylib功能

#### 使用工具提取信息：
```bash
# 1. 提取字符串
strings srjsjlbpro.dylib | grep -i "vip\|pro\|premium"

# 2. 查看依赖
otool -L srjsjlbpro.dylib

# 3. 导出Objective-C类
class-dump -H srjsjlbpro.dylib -o headers/

# 4. 反汇编分析
objdump -d srjsjlbpro.dylib > disasm.txt
```

#### 关键信息提取：
```
从dylib中找到：
✓ 目标应用Bundle ID
✓ Hook的类名和方法名
✓ 加密/解密方法
✓ 特殊功能（如清理缓存）
```

### 步骤2: 确定转换策略

#### 策略A: 网络请求修改（适用于Loon）
```
适用场景：
✓ VIP状态由服务器返回
✓ 应用会定期检查订阅状态
✓ 没有严格的本地验证

实现方式：
1. 抓包分析API请求
2. 找到返回VIP状态的接口
3. 编写Loon脚本修改响应
```

#### 策略B: 本地方法Hook（需要Frida）
```
适用场景：
✓ VIP状态存储在本地
✓ 应用有严格的本地验证
✓ 不依赖网络请求

实现方式：
1. 使用Frida注入
2. Hook相关Objective-C方法
3. 修改返回值为true
```

### 步骤3: 抓包分析API

#### 使用Charles/Fiddler：
```
1. 配置代理和证书
2. 打开目标应用
3. 捕获所有请求
4. 筛选包含 vip, pro, premium 的请求
5. 分析请求和响应结构
```

#### 典型的VIP检查请求：
```
请求：
GET https://api.maleworkout.com/v1/user/profile
Headers:
  Authorization: Bearer xxx
  Content-Type: application/json

响应：
{
  "data": {
    "userId": "12345",
    "isVip": false,        ← 关键字段
    "isPro": false,        ← 关键字段
    "vipExpireTime": null
  }
}
```

### 步骤4: 编写Loon插件

#### 创建.plugin文件：
```ini
#!name=应用名称解锁
#!desc=功能描述
#!author=作者
#!icon=图标URL

[Script]
http-response 正则表达式URL script-path=脚本URL,requires-body=true, timeout=10, tag=标签名

[MITM]
hostname = 域名列表
```

#### 创建.js脚本：
```javascript
let url = $request.url;
let body = $response.body;

try {
    let obj = JSON.parse(body);
    
    // 根据实际响应结构修改
    if (obj.data) {
        obj.data.isVip = true;
        obj.data.isPro = true;
        obj.data.isPremium = true;
    }
    
    body = JSON.stringify(obj);
} catch (e) {
    console.log('Error: ' + e);
}

$done({body});
```

## 🎯 实际转换案例

### 案例: MaleWorkout插件转换

#### 原始dylib分析结果：
```
目标应用: com.abishkking.maleworkout
Hook方法:
- isVip → 返回true
- isPro → 返回true
- isPremium → 返回true

额外功能:
- 清理缓存
- 清理UserDefaults
- 清理Keychain
```

#### 转换为Loon插件：

##### 1. 抓包确认API
```
发现关键接口：
https://api.maleworkout.com/v1/user/profile
https://api.maleworkout.com/v1/subscription/status
```

##### 2. 分析响应结构
```json
{
  "data": {
    "user": {
      "id": "xxx",
      "isVip": false,
      "isPro": false
    },
    "subscription": {
      "type": "free",
      "isActive": false
    }
  }
}
```

##### 3. 编写Loon脚本
```javascript
let obj = JSON.parse($response.body);

if (obj.data) {
    if (obj.data.user) {
        obj.data.user.isVip = true;
        obj.data.user.isPro = true;
    }
    
    if (obj.data.subscription) {
        obj.data.subscription.type = "premium";
        obj.data.subscription.isActive = true;
    }
}

$done({body: JSON.stringify(obj)});
```

##### 4. 配置.plugin文件
```ini
#!name=MaleWorkout健身解锁
#!desc=解锁MaleWorkout Pro/VIP功能

[Script]
http-response https?:\/\/.*maleworkout.*\/api\/(user|subscription) script-path=maleworkout.js,requires-body=true, timeout=10, tag=MaleWorkout

[MITM]
hostname = api.maleworkout.com, *.maleworkout.com
```

## ⚠️ 转换限制

### 无法转换的功能

#### 1. 本地数据操作
```
dylib功能:
- cleanCache (清理缓存)
- clearUserDefaults (清理UserDefaults)
- clearKeyChain (清理Keychain)

Loon限制:
❌ 无法直接操作应用数据
✓ 需要用户手动在应用内操作
```

#### 2. 本地验证绕过
```
dylib功能:
- Hook本地验证方法
- 修改内存中的标志位

Loon限制:
❌ 无法修改应用内存
✓ 只能修改网络请求
✓ 需要配合Frida使用
```

#### 3. 加密/解密功能
```
dylib功能:
- BEncryptString (加密)
- BDecryptString (解密)

Loon限制:
❌ 无法调用应用内的加密方法
✓ 需要自己实现加密算法（如果需要）
```

### 需要额外处理的情况

#### 1. 证书绑定 (Certificate Pinning)
```
问题: 应用验证服务器证书
解决: 
- 越狱设备: 使用SSL Kill Switch 2
- 非越狱设备: 需要重打包应用
```

#### 2. 签名验证
```
问题: 应用验证响应签名
解决:
- 分析签名算法
- 在脚本中实现签名
- 或使用Frida Hook验证方法
```

#### 3. 时间戳验证
```
问题: 应用检查时间戳是否过期
解决:
- 在脚本中修改时间戳
- 设置为未来时间
```

## 🔧 高级转换技巧

### 技巧1: 多层验证绕过

#### 场景：应用同时检查多个字段
```javascript
// 完整的VIP状态修改
if (obj.data) {
    // 第一层：用户基本信息
    obj.data.isVip = true;
    obj.data.isPro = true;
    obj.data.isPremium = true;
    obj.data.subscriptionType = "premium";
    
    // 第二层：订阅信息
    if (obj.data.subscription) {
        obj.data.subscription.isActive = true;
        obj.data.subscription.type = "premium";
        obj.data.subscription.startDate = "2020-01-01";
        obj.data.subscription.endDate = "2099-12-31";
    }
    
    // 第三层：权限列表
    if (obj.data.permissions) {
        obj.data.permissions = {
            advancedWorkouts: true,
            customPlans: true,
            noAds: true,
            offlineMode: true
        };
    }
    
    // 第四层：功能开关
    if (obj.data.features) {
        for (let key in obj.data.features) {
            obj.data.features[key] = true;
        }
    }
}
```

### 技巧2: 动态响应生成

#### 场景：需要根据请求参数返回不同内容
```javascript
let url = $request.url;
let params = new URL(url).searchParams;
let userId = params.get('userId');

let obj = {
    data: {
        userId: userId,
        isVip: true,
        isPro: true,
        vipExpireTime: "2099-12-31T23:59:59Z"
    }
};

$done({body: JSON.stringify(obj)});
```

### 技巧3: 条件性修改

#### 场景：只修改特定用户的VIP状态
```javascript
let obj = JSON.parse($response.body);

// 只为测试账号解锁
if (obj.data && obj.data.userId === "test123") {
    obj.data.isVip = true;
    obj.data.isPro = true;
}

$done({body: JSON.stringify(obj)});
```
